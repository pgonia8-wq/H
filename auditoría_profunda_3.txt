  🚨 #1 [CRÍTICO] — sell.mjs
  ───────────────────────────────────────────────────────────────
  Problema: Holdings update (L107) uses upsert WITHOUT OCC — race condition on holdings

  Tipo: concurrencia

  Cómo explotarlo:
    Token OCC (L93) passes, but two sells can read same heldAmount=100, both pass OCC on tokens table, then both upsert holdings with newAmount=50. User sells 100 tokens total but holdings shows 50.

  Impacto real:
    User can sell MORE tokens than they own by sending concurrent requests. The OCC is on the tokens table (circulating_supply) but NOT on holdings.

  Evidencia en código:
    L107: supabase.from("holdings").upsert({...amount: newAmount...}) — no .eq("amount", heldAmount)

  Solución concreta:
    Use atomic: UPDATE holdings SET amount = amount - tokensToSell WHERE amount >= tokensToSell, or add .eq("amount", heldAmount) check
